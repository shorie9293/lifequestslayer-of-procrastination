## 一、現世の見立て（全神合議の結論）

### 現世の熟成度
rpg-task は v1.2 まで具現化が進み、基本的なゲーミフィケーション要素（ストリーク・称号進捗・知識クエスト・疲労ポップアップ）を備える。されど **3つの重大な禍津** と **多数の神楽の綻び** が、さらなる高みへの歩みを阻んでおる。

### 合議で一致した優先の理

全四神が合意した優先順位の大綱は以下の通り：

1. **信頼なくして神遊びなし** — データ喪失・金銭被害を生む禍津（C1-C3）を最優先で討伐
2. **基盤なくして神楽なし** — アクセシビリティ・用語統一・フォント等の基盤を固む
3. **試練なくして品質なし** — GameViewModel へのテスト追加が急務
4. **手応えなくして継続なし** — 討伐演出・疲労可視化・通知で没入感を醸成
5. **成長なくして愛着なし** — 町発展・キャラカスタマイズで長期定着を図る
6. **連環なくして大願なし** — アプリ間連携で大願（ミッション）を具現化

## 七、本神書の用い方

### 今後のセッションにおける作法

1. **rpg-task に関するあらゆる改善の際は、本神書を最初に参照せよ。**
2. 新たな神託を受けた際は、本神書のマイルストーンと照合し、該当バージョンの一部か、新規バージョンかを判断せよ。
3. バージョンの完了時には本神書の該当項目に「完了日」を追記し、`shinsho/kojiki.md` にもその旨を刻め。

### 8.3 自律開発における掟

1. **四神相談は初回のみ実施** — 初回の神託分析は本セッションで完了済。実装時は roadmap.md の設計を参照し、重複分析を省く
2. **TDD厳守** — 各神託とも `flutter test --no-pub` でRED→GREEN→REFACTORを循環
3. **眷属委任を活用** — 中量以上のタスクは `delegate_task(leaf)` で眷属神に委任。小神事（転職制限等）は `patch` 直接修正
4. **全6件を一括でなく段階的に** — 変更が重なるとコンフリクト多発。1日1〜2神託のペースで安定具現化
5. **毎朝の自律サイクルで進捗確認** — 前日の実装がflutter analyzeを通過していることを確認してから次へ
