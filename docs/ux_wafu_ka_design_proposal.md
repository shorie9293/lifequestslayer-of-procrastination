# 第一項：画面遷移アニメーション（和風トランジション）🟠P1

## 現状の問題
- `PageView` による横スライド（デフォルトのMaterialスライド）
- 和風らしさゼロ
- 遷移時の演出なし

## 提案：4種の和風トランジション

### 1.1 メインタブ遷移（PageView横移動）— 「障子開き」

```
[実装方式]
AnimatedSwitcher または PageView のカスタム transitionBuilder を差し替え

[アニメーション詳細]
- 前画面が「スッ」と奥に縮みながら leftSide から fade
- 次画面が「パッ」と手前に拡大しながら rightSide から出現
- 縦の桟（フレーム）が一瞬オーバーレイ
- 持続時間: 350ms（easeInOutCubic）
- カーブ: Curves.easeInOutCubic（和の「間」を感じさせる緩急）

[Flutter実装イメージ]
class ShojiTransition extends StatelessWidget {
  Widget build(BuildContext context, Widget child, Animation<double> animation) {
    return SlideTransition(
      position: Tween<Offset>(
        begin: const Offset(0.3, 0),
        end: Offset.zero,
      ).animate(CurvedAnimation(
        parent: animation,
        curve: Curves.easeInOutCubic,
      )),
      child: FadeTransition(
        opacity: Tween<double>(begin: 0.0, end: 1.0).animate(animation),
        child: child,
      ),
    );
  }
}
```

### 1.2 ダイアログ開閉 — 「巻物開き」

| 項目 | 内容 |
|------|------|
| 開く | Centerから横方向に「くるる」と巻物が広がるように ScaleX 0→1 |
| 閉じる | 右端から「くるり」と巻物が閉じるように ScaleX 1→0 |
| 持続時間 | 400ms（少しゆっくり） |
| カーブ | Curves.easeOutBack（巻物の反動を表現） |

### 1.3 通知/スナックバー — 「神託の紙片」

- 上から障子の隙間を滑るように「ひらり」と紙片が降りてくる
- 背景が半透明の金色（#D4A038 @20% opacity）
- 持続時間: 300ms

### 1.4 レベルアップ演出 — 「神階昇天」

```
[既存の showGeneralDialog + スケールアニメーションを和風化]

現状: Transform.scale(Curves.elasticOut) ← これは和風として悪くないが、色味が西洋的

改修:
1. 背景に金色の光芒が同心円状に広がる（CustomPainter）
2. 中央に「巻物」が開き、階位が墨文字で現れる
3. その場で1秒ほど祝福の効果音（和太鼓1発）
4. 紙吹雪ではなく「御神籤（おみくじ）」が舞い散る
```

### 実装の手順

```
Step 1: lib/core/theme/transitions/ を作成
Step 2: wafu_transitions.dart に上記4種を実装
Step 3: main_screen.dart の PageView に shoji_transition を組み込み
```

# 第七項：アクセシビリティとSemantics 🟠P1

## 7.1 基本方針

> **Semanticsラベルは「和風化しない」が「神話的補足」を追加する**

### 理由
1. **スクリーンリーダー互換性**: 音声読み上げは平板な現代日本語が最適
2. **国際化**: 将来的に英語対応した時、和風ラベルが障害になる
3. **認知負荷**: 視覚障害者が「修練場」を「しゅれんじょう」と聞いても概念理解にワンクッション必要

### 実装方針

```dart
// ✅ 良い例
SemanticHelper.interactive(
```

## 7.2 アクセシビリティ対策一覧

| # | 対策 | 優先度 | 実装方法 |
|---|------|-------|---------|
| 1 | **色＋形の二重表現** | 🔴 P0 | ランク「天/地/人/凡」に色＋アイコン＋文字 |
| 2 | **アニメーション速度調整** | 🟠 P1 | `AnimationController` の speed を設定で変更可能に |
| 3 | **和風エフェクトOFFモード** | 🟠 P1 | 設定トグル「和風演出を簡略化」→ フラットなMaterial表示に |
| 4 | **フォント拡大** | 🔴 P0 | 既存の `fontSizeScale` 継続。和風フォントでも崩れないレイアウト |
| 5 | **フォントコントラスト** | 🟠 P1 | 和紙色背景に薄墨文字はコントラスト比4.5:1を確保 |
| 6 | **タップ領域** | 🔴 P0 | 44px以上維持（既存のMaterialデフォルトから変更なし） |
| 7 | **震えを誘発する演出** | 🟠 P1 | パーティクル・激しい画面揺れは設定でOFF可能に |
| 8 | **和風フォント代替** | 🟡 P2 | Noto Serif JP が読みづらい場合、システム標準フォントにフォールバック |
| 9 | **色覚多様性モード** | 🟡 P2 | 赤/金/紫の区別がつかない場合、形＋アイコンで補完 |
| 10 | **VoiceOver/TalkBack対応** | 🔴 P0 | `Semantics` ラベルは常に現代日本語。既存の `SemanticHelper` 継続利用 |

## 7.3 和風UIにおけるアクセシビリティ上の注意点

```
【注意1】和紙色背景 #F5F0E8 に薄墨文字 #333333
  → コントラスト比 11.6:1 ✅（基準4.5:1をクリア）

【注意2】金 #D4A038 に白文字 #FFFFFF
  → コントラスト比 1.4:1 ❌ 読めない！
  → 対策: 金色ボタンの文字は深紫 #1A1040 か焦茶 #5D3A1A に

【注意3】深紫 #1A1040 に朱 #E85050
  → コントラスト比 3.9:1 ⚠️ 基準ぎりぎり
  → 対策: 重要でない装飾に限定。テキストとして使わない

【注意4】和風パターン（市松/麻の葉）を背景に使う場合
  → パターン上にテキストを重ねない
  → どうしても重ねる場合はパターン不透明度5%以下＋テキスト背後にソリッドシャドウ
```
