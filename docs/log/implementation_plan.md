# 疲労度・キャラ・町の可視化実装計画（エージェント会議議事録）

## 🎯 議題：疲労度、キャラメイキング、町のメイキングの可視化はどこから手を付けるべきか？

### 📝 Step 1: Brainstorming（ブレインストーミング）
**🎮 GamingExpert**（面白さ最優先のRPGマニア）
> 「可視化！これこそRPGの醍醐味だ！！
> まずは絶対に**『キャラの見た目』**からだ！自分が強くなっている実感、かっこいい装備を手に入れた実感が湧かないとモチベーションが続かない。装備したスキンがホーム画面にドカンと表示される仕組みから作ろうぜ！
> 次に**『町』**だ。テントから城に発展していく過程を背景画像で見せることで、日々のタスクが「領地開拓」になるんだ！
> 疲労度は…画面の隅でいい。マイナス要素というより、「今は宿屋で休むべき時間だ」というロールプレイを促す演出にすべきだな！」

**💼 ProductivityExpert**（効率最優先の仕事効率化オタク）
> 「ちょっと待ってくれ。これは基盤がTODOアプリだ。最も重要なのは『タスク実行の効率と正確なフィードバック』だ。
> 優先すべきは文句なしで**『疲労度の可視化』**だ。現在、疲労による報酬減衰（バフ低下）が内部で起きているのにユーザーに伝わっていない。これはタスク管理ツールとして致命的だ！ユーザーが「今日はあとどれだけタスクをこなせるか」を計算できるように、ヘッダー部分に明確なゲージと警告アイコンを早急に追加すべきだ。
> キャラや町の見た目変更は、UI改修コストも高く、単なる趣味の領域だ。後回しで十分！」

### ⚖️ Step 2: Mediation（意見調整）
**🤝 BalanceMediator**（冷静なプロダクトマネージャー）
> 「両者の意見はどちらも一理あるね。
> ProductivityExpertの言う通り、現在の『疲労ペナルティが隠れている状態』はUXとして大きなマイナスなので、早急な解決が必要だ。幸い、ヘッダーへのUI追加だけで済むため実装コストも低く、ここを**第1フェーズ**とするのが最も筋が良い。
> しかし、GamingExpertが言うように、ご褒美としての『見た目の変化（町・キャラ）』はアプリの長期的な継続（リテンション）において極めて重要だ。
> まずは低コストで大きなUX改善になる疲労度を片付け、次に中コストで達成感を出せる「町の背景変化」、最後にデータ構造の変更を伴う「キャラスキン装備システム」へと進めるのが、実用性とゲーム性のバランスが最も良いアプローチだろう。」

## 📋 決定事項：実装の優先順位とフェーズ

会議の結果、**「1. 疲労度の可視化」**から着手し、実用性の課題をクリアした上で、ゲーミフィケーション要素の拡充（町→キャラ）に進むのが最適という結論に至りました。

### Phase 1: 疲労度の可視化（優先度：高 / コスト：低）
現在内部で計算されている「疲労ペナルティ」をユーザーに明示します。
- **改修対象**: `PlayerStatusHeader`
- **内容**: ヘッダー（金貨やレベルの横）に、現在の疲労状態（😄元気 / ⚠️疲労 / 💀過労）と、あと何回タスクをこなせるかの目安ゲージを追加。

### Phase 2: 町の発展（拠点）の可視化（優先度：中 / コスト：中）
日々のタスク消化が「自分の陣地の拡大」に繋がる演出を強化します。
- **改修対象**: `TownScreen`
- **内容**: `TownScreen`の背景画像（現在固定）を、購入済みの「拠点拡張アイテム（テント〜ギルドハウス）」の最高レベルに合わせて動的に切り替える処理を追加。

### Phase 3: キャラメイキングの可視化（優先度：低 / コスト：高）
プレイヤーの最終的なやり込みと自己表現の場を提供します。
- **改修対象**: `Player`モデル、`TownScreen`（ショップ）、メイン画面・ヘッダー等
- **内容**: 
  - `Player`モデルに`equippedSkin`（装備中のスキン）を保存する項目を追加。
  - ショップで購入したアイテムを「装備」できる仕組みを導入。
  - 装備中の職業とスキンを反映したテキストまたは簡易アイコン（アバター）を画面上に表示。
