# システム設計・変更履歴：可視化機能（疲労度・町・キャラ）

## 1. 概要
ユーザーのモチベーション維持（ゲーミフィケーション要素）と、システム状態の明確化（タスク管理ツールとしての利便性）を両立させるため、以下の3つの要素の「可視化」を実装した。

1. **疲労度（体調）の可視化**
2. **拠点（はじまりの街）の発展の可視化**
3. **キャラクター（スキン）の可視化**

---

## 2. 企画会議録（エージェント・ディスカッション）

実装優先順位の決定にあたり、`@agent.yaml` の枠組みを用いたエージェント会議を実施した。

### ブレインストーミング
*   **GamingExpert（ゲーム性重視）**: 「着手すべきは『キャラの見た目』と『町の発展』だ！強くなる実感、陣地が拡大する実感がRPGの醍醐味だ。疲労度はマイナス要素なので画面の隅にあればいい」
*   **ProductivityExpert（実用性重視）**: 「待ってくれ、これはTODOアプリだ。現在内部で発生している『疲労による報酬減衰』がユーザーに見えていないのは致命的。最優先は『疲労度の可視化ゲージ』の追加だ」

### 意見調整・裁定
*   **BalanceMediator（バランス調整）**: 「UXの観点からペナルティの可視化（疲労度）は急務であり、UI追加だけで済むためコストも低い。これを第1フェーズとする。一方で長期継続にはご褒美（町・キャラ）も必須なので、これらを第2・第3フェーズとするのが最適」
*   **DecisionMaker（最終決定）**: 上記の調停案を採択し、以下の順序で実装を進めることとした。

---

## 3. 実装内容

### Phase 1: 疲労度の可視化
内部で計算されていた疲労ペナルティ（本日の限界タスク数超過による報酬減衰）をユーザーに明示した。
- **UI**: ヘッダー（`PlayerStatusHeader`）に、「体調ステータス（😄元気 / ⚠️疲労 / 💀疲労困憊）」のアラートと、限界までの進捗ゲージを追加。
- **効果**: 「今日はあとどれだけタスクができるか」の目安がつき、過労状態の把握が容易になった。

### Phase 2: 町の発展（拠点）の可視化
所持している拠点アイテムに応じて、町の背景が変化するようにした。
- **UI**: `TownScreen`の背景を、所持している最高ランクの拠点アイテム（「小さなテント」〜「ギルドハウス」）を判定し、背景色やアイコンを動的に切り替える仕組みを追加。
- **効果**: 日々のタスク消化が「自分の陣地の拡大」に繋がっているという達成感をもたらした。

### Phase 3: キャラメイキング（装飾）の可視化
キャラクターの着せ替え機能を実装した。
- **データ構造**: `Player`モデルに `equippedSkin`（装備中のスキンID）を追加し、永続化（Hive）に対応。
- **UI**: 
  - `TownScreen`のショップにて、購入済みアイテムの「装備する / 外す」ボタンを実装。
  - `PlayerStatusHeader`の左端に、装備中スキンに対応する「アバターアイコン（🧥、🎖️、🪄、👑など）」を表示。
- **効果**: 自身の分身の見た目をカスタマイズできるようになり、やり込みのモチベーションが強化された。
