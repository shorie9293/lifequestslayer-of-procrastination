## 序：七つの神託とは何か

> **神託（しんたく）**——神より授かる予言の言葉。
> この現世においては、**創造主様（ユーザー）の行動を予告し、導き、報いる仕組み**を指す。
>
> 七つの神託はそれぞれ異なる「心の動き」を司る：
> - **驚き**（意外な報酬・予告）
> - **納得**（仕組みへの理解・ルールの透明性）
> - **達成感**（目標達成・成長実感）
>
> 此れを神楽の舞として組み立て、ユーザーの日常にリズムと彩りを添える。

## 第一の神託：暁の神託（Dawn Oracle）— 日々の訪れの予言 🔴

### 概要
日付が変わった最初の起動時、本日の運勢を「神託」として表示する。
既存の**ストリーク機構**と**ログインボーナス**を統合し、予告形式に昇華する。

### 表示画面／タイミング
- **画面**: HomeScreen（MainScreen 最上部）、起動直後に「巻物開き」ダイアログ
- **タイミング**: `_checkMissions(login: true)` の直後、`pendingStreakReward` / `pendingLoginBonusAmount` がセットされている時
- **表示優先度**: チュートリアル > 暁の神託 > 通常画面

### UI表現

```
┌──────────────────────────────────────────┐
│                                          │
│   ╔══════════════════════════════╗       │
│   ║      【暁の神託】            ║       │
│   ║                              ║       │
│   ║  今日は 七日目 の訪れなり    ║       │
│   ║  神よりの報い：              ║       │
│   ║  金貨 +1,000 🪙             ║       │
│   ║  宝珠 +20 💎                ║       │
│   ║                              ║       │
│   ║   ┌──────────────────┐      ║       │
│   ║   │  神託を受け取る  │      ║       │
│   ║   └──────────────────┘      ║       │
│   ║                              ║       │
│   ║  今日の導き：               ║       │
│   ║  「Bランクの依頼が吉」     ║       │
│   ╚══════════════════════════════╝       │
│                                          │
│  （背景には和風の雲文様＋朝日）          │
└──────────────────────────────────────────┘
```

### 感情曲線設計

| フェーズ | 感情 | 演出 |
|---------|------|------|
| ① 出現 | 😲 **驚き** | 画面中央に巻物が「スッ」と現れる。背景が暗転し、朝日のような金色の光芒 |
| ② 発見 | 🤔 **好奇心** | まず「暁の神託」のタイトルが墨文字で滲むように表示。続いてストリーク日数 |
| ③ 認識 | 😊 **納得** | 報酬内容が金貨→宝珠の順に一文字ずつ出現（タイプライター風） |
| ④ 獲得 | 🎉 **達成感** | 「神託を受け取る」ボタンを押すと、金貨のチャリン音＋XPバーのアニメーション |
| ⑤ 導き | 🤗 **期待感** | 「今日の導き」テキストで、今日のタスク傾向を予告 |

### 既存メカニクスとの整合性

| 要素 | 整合方法 |
|------|---------|
| `StreakService.checkAndUpdateStreak()` | 神託の内容はこの結果を元に生成。ストリーク日数に応じて報酬テーブルを参照 |
| `pendingStreakReward` / `pendingLoginBonusAmount` | これらを神託文に変換して表示。受け取り後に従来通りクリア |
| ストリーク報酬テーブル（rpg-shinso.md） | 3日・5日・7日・14日・30日の閾値を神託文に反映 |

### 実装の最小単位（v1.3〜v1.4 即時対応可能）
- 既存の `pendingStreakReward` を活用：nullでなければ暁の神託ダイアログを表示
- `assets/data/dawn_oracle_messages.json` にストリーク日数ごとの神託文を外部化
- スクナヒコナ（既存設計）と併用：神託の語り手として彼が登場しても良い

## 第二の神託：戦の神託（Battle Oracle）— 連撃の予言 🟠

### 概要
タスク完了時、連続完了（コンボ）が発生した際に「戦の神託」が発動。
「次の一撃は強化される」と予告し、ユーザーに連続完了の動機を与える。

### 表示画面／タイミング
- **画面**: BattleScreen（修練場）の TaskCard 下部、または戦果報告書の下部
- **タイミング**: `completeTask()` 成功後、コンボカウントが特定の閾値（3/5/7/10）に達した時
- **既存要素の活用**: `Player.comboCount` を参照

### UI表現

```
┌──────────────────────┐
│  📜 戦の神託          │
│                      │
│  「三連撃、成る！」  │
│                      │
│  次の一撃は          │
│  XP +50%  UP！       │
│                      │
│  ────🔥🔥🔥────      │
│  コンボ: 3連続       │
└──────────────────────┘
```

連続数に応じて「連撃」の文字が変化：
| コンボ数 | 神託文 | ボーナス | エフェクト |
|---------|--------|---------|-----------|
| 3 | 「三連撃、成る！」 | XP +20% | 小さな炎がメラメラ |
| 5 | 「五連撃、冴える！」 | XP +50% + 金貨ボーナス | 炎が大きくなる |
| 7 | 「七連撃、神速なり！」 | XP +100% | 雷光エフェクト |
| 10 | 「十連撃、天に轟く！」 | XP +200% + 宝珠ボーナス | 画面全体に閃光 |

### 感情曲線設計

| フェーズ | 感情 | 演出 |
|---------|------|------|
| ① 閾値到達 | 😲 **驚き** | タスク完了のエフェクト中に金色の文字がフラッシュ |
| ② 神託表示 | 🤩 **高揚** | 画面中央に「戦の神託」の巻物が一瞬で展開。漢数字が力強く表示 |
| ③ 報酬予告 | 😏 **期待感** | 次のタスクに取り組む際、TaskCard に「⚡ 神託活性中」のバッジ表示 |
| ④ 神託達成 | 🎊 **大達成感** | ボーナスXPが加算される瞬間、通常より派手なパーティクル演出 |

### 既存メカニクスとの整合性

| 要素 | 整合方法 |
|------|---------|
| `Player.comboCount` | 既存フィールド。comboCount が 3/5/7/10 に達した時に発火 |
| `TaskCompletionService` | ボーナスXPの計算は同サービスの `complete()` 内で行う |
| 戦果報告書ダイアログ（UX-6） | 戦の神託は報告書内のセクションの一つとして統合表示可能 |
| パーティクルエフェクト（v1.4） | 「十連撃」時は通常パーティクルより3倍の密度で演出 |

## 第三の神託：叡智の神託（Wisdom Oracle）— 知識の導き 🟠

### 概要
既存の**知識クエスト（Knowledge Quest）**を「叡智の神託」として再演出。
どのような問題が出るかを事前に「神託（ヒント）」として表示することで、
「勉強する→問題に答える」の能動的サイクルを創出する。

### 表示画面／タイミング
- **画面**: GuildScreen（寄合所）— 依頼書を「受託」した直後
- **タイミング**: `acceptTask()` 成功後、対象タスクのカテゴリに関連する知識クエストがあれば予告表示
- **既存要素の活用**: `_knowledgeQuests` リスト、`KnowledgeQuest.category`

### UI表現

```
[タスク受託直後]
┌──────────────────────────────────┐
│  📜 叡智の神託                  │
│                                  │
│  「この依頼には叡智の試練が     │
│    待ち構えておるぞ」           │
│                                  │
│  兆し: 📖 生産性                │
│  難易度: ★★☆☆☆                 │
│  挑む?: [挑む] [後で]            │
└──────────────────────────────────┘
```

- **「挑む」**: 従来通りタスク完了時に30%抽選。ただしカテゴリが一致する問題が出やすくなる（50%まで上昇）
- **「後で」**: 抽選確率を通常の30%に維持
- **兆し**: 問題のカテゴリのみを予告（具体的な問題文は見せない）

### 感情曲線設計

| フェーズ | 感情 | 演出 |
|---------|------|------|
| ① 受託直後 | 🤔 **好奇心** | 依頼書をめくると裏面に「叡智の神託」が現れる |
| ② 予告表示 | 🧐 **期待感** | カテゴリ名が墨文字で滲む。ユーザーは「どんな問題か」を想像する |
| ③ タスク完了時 | 🎯 **集中** | 問題が出た時、「神託は当たった」という納得感（予告カテゴリと一致） |
| ④ 正解時 | 🧠 **知的達成感** | 「叡智の神託、成就！」のテキスト＋ボーナスXPに金色の+α表示 |

### 既存メカニクスとの整合性

| 要素 | 整合方法 |
|------|---------|
| `drawKnowledgeQuest()` | 変更なし。予告機能はacceptTask()のタイミングで別途表示 |
| `KnowledgeQuest.category` | 予告に使用。`assets/data/knowledge_quests.json` の構造は変更不要 |
| 30%抽選ロジック | 維持。「挑む」選択時のみ該当カテゴリの出現率を50%にバイアス |
| `bonusMessages` | 「📚 叡智の試練 正解！」のメッセージに「神託成就」の冠詞を追加 |

## 第四の神託：運命の神託（Fate Oracle）— 籤引きの狂喜 🟡

### 概要
一日一回、町（門前町）の「社」の一角で「運命の神籤（おみくじ）」を引ける。
大吉〜凶までの結果に応じて、金貨・宝珠・一時的なバフ効果が得られる。
**完全に任意**のインタラクション——押し付けない神託。

### 表示画面／タイミング
- **画面**: TownScreen（門前町）→ 新設「神籤所（おみくじどころ）」セクション
- **タイミング**: 一日一回、いつでも任意で実行可能
- **追加場所**: TownScreen の「宿屋」と「宝石ショップ」の間に新セクション追加

### UI表現

```

│  🎴 運命の神籤                  │
│                                  │
│   ┌──── 籤箱 ────┐              │

│   │  [ ？  ]      │              │
│   │  [ ？  ]      │              │
│   │  [ ？  ]      │              │
│   └───────────────┘              │
│                                  │
│  今日の残り回数: 1回             │
│                                  │
│  [ 神籤を引く ]                  │
└──────────────────────────────────┘

[結果表示]
┌──────────────────────────────────┐
│  🎴 ==== 運命の神託 ====        │
│                                  │
│      ╔══════════════╗           │
│      ║  末吉         ║          │
│      ║  (まずまず)   ║          │
│      ╚══════════════╝           │
│                                  │
│  金貨 +50 🪙 を授けたり        │
│                                  │
│  ──── 神託のことわり ────      │
│  大吉: 金貨+500 / 宝珠+10      │
│  吉:   金貨+200 / 経験値+20%UP  │
│  中吉: 金貨+100                 │

```

**確率テーブル**:
| 結果 | 確率 | 報酬 | 追加効果 |
|------|------|------|---------|
| 大吉 | 5% | 🪙+500 💎+10 | 翌日の全タスクXP+30% |
| 吉 | 15% | 🪙+200 | 次のタスクXP+20% |
| 中吉 | 30% | 🪙+100 | — |
| 末吉 | 30% | 🪙+50 | — |
| 凶 | 15% | 🪙+10 | 「明日は運気上昇」ポジティブな予告 |
| 大凶 | 5% | 💎+5 | 「不運の後の好機」としてストリーク保護効果（1日スキップしても継続扱い） |

### 感情曲線設計

| フェーズ | 感情 | 演出 |
|---------|------|------|
| ① 籤箱を開く | 🤩 **期待と高揚** | 和風の箱が「ガラリ」と開くアニメーション。BGMが一瞬止まる |
| ② 引き出す | 😬 **緊張** | 籤がゆっくりと引き出される。画面に細かい振動 |
| ③ 結果表示 | 🎭 **驚き＋納得** | 大吉/凶でまったく異なる演出（大吉は金の光芒、凶は静かな雨粒エフェクト） |
| ④ 報酬受取 | 🎉 **達成感／笑い** | 凶でも「明日はある」という前向きメッセージで不快感ゼロに |

### 既存メカニクスとの整合性

| 要素 | 整合方法 |
|------|---------|
| `Player.coins` / `Player.gems` | 報酬として加算 |
| バフ効果 | 新設 `Player.oracleBuffExpMultiplier`（翌日リセットの一時的バフ値）を Player モデルに追加 |
| `completeTask()` | バフ値が設定されている場合、XP計算に乗算 |
| ストリーク保護（大凶） | `_checkAndUpdateStreak()` 内で保護フラグを参照。`Player.streakProtection` 追加 |
| 一日一回制限 | `settingsRepository` に `lastOracleDate` を保存。既存の `lastLoginDate` 保存パターンと同様 |

## 第五の神託：階位の神託（Rank Oracle）— 成長の予言 🟠

### 概要
称号の解放条件が近づいた時、あるいはレベルアップが目前の時、
「階位の神託」が予告として表示される。

### 表示画面／タイミング
| トリガー | 表示場所 | タイミング |

| 称号解放まで残り10% | TownScreen（称号セクション） | タブ移動時、またはタスク完了後 |

| 職業熟練度（マスター）まで残り1レベル | TempleScreen（社） | 職業選択画面を開いた時 |

### 感情曲線設計

| フェーズ | 感情 | 演出 |
|---------|------|------|
| ① 接近検知 | 🤨 **気づき** | 進行度バーに金色の輝きが走る。数値が通常より大きく表示 |

| `Player.addExp()` / `expToNextLevel` | 残りEXPが `expToNextLevel * 0.2` 以下で発動 |
| `LevelUpDialog` | 神託の「成就」はレベルアップダイアログに統合。「神託成就」のテキストを追加 |

週の初め（月曜）と月の初め（1日）に、その期間の「星の配置」を神託として表示。

### 目標生成ロジック（簡易ルールベース）

週の目標（3つ中2つをランダム選択）:
1. `Bランク依頼をN件成就せよ`（N = max(3, player.level ~/ 5)）
2. `連続ログインを維持せよ`
3. `獲得XP合計XXXX以上`（前週の実績×1.2）
4. `サブタスクをN件完了せよ`
5. `知識クエストにN回正解せよ`

月の目標（3つ固定）:
1. `合計30件の依頼を成就せよ`
2. `位階を2つ上げよ`
3. `新たな称号を1つ得よ`

### 感情曲線設計

## 根本思想書の五戒との整合性確認

| 五戒 | 内容 | 7神託の遵守状況 |
|------|------|----------------|
| **感情操作禁止** | ユーザーの不安を煽って行動させない | ✅ 「凶」でもポジティブな締めくくり。「大凶」はストリーク保護というギフトに変換。不安を煽る表現は一切なし |
| **課金依存禁止** | 課金しないと遊べない設計禁止 | ✅ 運命の神託の籤引きは無料。宝珠消費は終焉の神託の「選択肢の一つ」に過ぎず、宿屋や「今日はここまで」でも同等の体験を得られる |
| **中毒性設計禁止** | やめたくてもやめられない設計禁止 | ✅ 各神託は1日の出現回数が厳格に制限されている。連鎖効果も「気づき」レベルで、強制ループは設計されていない |
| **虚偽の期待禁止** | 実際の効能を誇張しない | ✅ 神託文は神話的装飾を含むが、報酬は明確な数値で表示。星の神託の「運勢」はゲーム内パラメータのみに影響し、現実の効能を偽らない |
| **孤独感の悪用禁止** | 「他者はやってる」と孤独感で煽らない | ✅ 他者比較の要素は一切なし。全ての神託は「自己の成長」と「自己のリズム」に焦点を当てている |
