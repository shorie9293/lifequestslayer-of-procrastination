## 一、現状分析

### 1.1 現行の職業とスキル体系

| 職業 | enum | 現マスタースキル | マスタリーLv | 常時/選択 |
|------|------|----------------|-------------|----------|
| **浪人（冒険者）** | `Job.adventurer` | ランク・枠数解放 | Lv10 | 常時ON |
| **侍** | `Job.warrior` | コンボボーナス（連続達成でEXP加算） | Lv14 | ON/OFF切替可 |
| **法師** | `Job.cleric` | 繰り返しクエスト（日次/週次） | Lv14 | ON/OFF切替可 |
| **陰陽師** | `Job.wizard` | サブタスク管理（全完了で討伐可） | Lv14 | ON/OFF切替可 |

### 1.2 現行の疲労システム

- 疲労警告: 5タスク超 → EXP/金 0.5倍
- 疲労重度: 10タスク超 → EXP/金 0.1倍
- 宿屋で休むと翌日の上限緩和
- **現状は「やりすぎペナルティ」**

### 1.3 現行のコンボシステム（侍のみ）

- タスク完了ごとに `comboCount++`
- ボーナス: `comboCount * 10` EXP加算
- 侍以外の職業ではコンボリセット
- **恩恵が分かりづらい**（例: 5コンボ=+50EXP。Bランク100EXPに対し微増）

### 1.4 現行の冒険者スキル（ランク・枠数解放）

`player.dart:108-136` の `questSlots` で:
- Lv1: B枠1
- Lv2: B枠2
- Lv5: A枠1, B枠3
- Lv10: S枠1, A枠2, B枠3
- 冒険者マスター時は他職でも冒険者Lvの枠数が適用される

## 二、創造主様の神託

### 神託①: 繰り返しタスク登録 → 冒険者のマスタースキルに
> 「繰り返しタスクの登録は、日課を作るのに重要なので、冒険者のマスタースキルとしたい」

**解釈**: 現状は法師のスキルだが、日課（ルーティン）作りは全冒険者の基本機能であるべき。冒険者は基本職なので、繰り返しタスク登録を冒険者の専権事項とする。

### 神託②: 連続実行ボーナス → 体力低時のボーナスに変更
> 「連続実行ボーナスはいまいち恩恵が分かりづらいので、体力が低いときにタスクこなすと経験値、金ともに上がるとかにしたい」

**解釈**: 現状の侍コンボボーナス（+10EXP×コンボ数）は微々たるもので恩恵が実感しづらい。代わりに、疲労が溜まっている状態（体力低）でのタスク討伐にボーナスを付与する。「減るほどに目が開く」逆転の理に合致する設計。

## 三、再設計案

### 3.1 新・職業スキル体系（案）

| 職業 | 新マスタースキル | 説明 | マスタリーLv |
|------|----------------|------|-------------|
| **浪人（冒険者）** | 🔄 **繰り返しクエスト登録** | 日次・週次・曜日指定の繰り返しクエストを作成可能に | Lv10（変更なし） |
| **侍** | 🔥 **底力ボーナス** | 疲労時（体力低）にEXP・金の獲得量が上昇。疲れるほど燃える | Lv14 |
| **法師** | 💰 **金運上昇 / レアドロップ率UP** | タスク討伐時の獲得金増加＋レアドロップ確率上昇 | Lv14 |
| **陰陽師** | 📋 **サブタスク管理**（変更なし） | サブタスク分割・全完了で討伐可能 | Lv14 |

### 3.2 侍「底力ボーナス」の詳細設計

**現状の疲労補正（ペナルティ）を逆転させる:**

| 疲労状態 | 現行 | 新（侍スキルON時） |
|---------|------|-------------------|
| 😊 快調 (0〜4タスク) | 1.0倍 | 1.0倍（変化なし） |
| 😐 やや疲れ (5〜9タスク) | **0.5倍** | **1.5倍** ✨ |
| 😓 疲労 (10+タスク, progress 0.7+) | **0.1倍** | **2.0倍** ✨ |
| 😵 限界 (progress 1.0, severe超え) | **0.1倍** | **3.0倍** ✨✨ |

- 侍以外の職業では現行通りのペナルティ（0.5倍/0.1倍）
- 侍スキルON時のみ逆転ボーナス発動
- 「疲れるほど強くなる」侍の美学

### 3.3 冒険者「繰り返しクエスト登録」の詳細設計

**現状の法師スキルを冒険者に移管:**

- 繰り返しクエスト（`RepeatInterval.daily/weekly` + 曜日指定）の作成権限を冒険者のマスタースキルに
- 法師で繰り返しクエストを作成済みの既存ユーザーは、そのまま使える（後方互換）
- 冒険者マスター前（Lv1〜9）は繰り返しクエスト作成不可
- 作成UIに「浪人Lv10で解放」のロック表示

### 3.4 冒険者の「ランク・枠数解放」の扱い

冒険者に2つのマスタースキルを持たせる選択肢：

| 案 | 内容 | 長所 | 短所 |
|----|------|------|------|
| **A** | 繰り返し登録をメインスキル、ランク枠数解放は常時パッシブに | 冒険者が2つの恩恵を持つ。基本職としての価値向上 | 複雑化 |
| **B** | ランク枠数解放を冒険者の常時パッシブに据え置き、繰り返し登録を追加のマスタースキルとして併存 | 既存バランス維持＋新機能追加 | 冒険者だけスキル2つは不公平感 |
| **C** | ランク枠数解放は全職共通の基本システムに降格、冒険者のマスタースキルは繰り返し登録のみに | 一貫性がある | 冒険者の既存恩恵が減るように見える |

**推奨: 案A** — 冒険者は基本職であり、ランク・枠数解放は「成長に伴う自然な拡張」として位置づけ、マスタースキル（明示的な固有スキル）は「繰り返しクエスト登録」とする。

### 3.5 法師の新スキル案

法師から繰り返しクエストを外す代わりの新マスタースキル案：

| 候補 | 内容 | 評価 |
|------|------|------|
| **💰 金運上昇** | 獲得コイン1.5倍 | シンプルで分かりやすい。回復・支援職として金銭面の恩恵 |
| **🍀 レアドロップ率UP** | レアドロップ確率2倍 | 運要素。法師の祈り・加護イメージに合う |
| **💤 疲労回復速度UP** | 宿屋の効果1.5倍 / 自然回復追加 | 回復職として本筋 |
| **🎯 デイリーミッションボーナス** | デイリー/ウィークリー達成報酬2倍 | 日々の継続を強化 |

**推奨: 💰金運上昇 + 🍀レアドロップUP の複合** — 「法師の加護により報酬が増す」というコンセプト。

### 4.2 実装ステップ

1. **疲労逆転ボーナス（侍）の実装** — `FatigueService` に `reverseFatigueMultiplier()` 追加 → `TaskCompletionService` で侍スキルON時に使用

4. **UI/チュートリアル更新** — 寺院画面・職業チュートリアルの文言を全更新
5. **テスト更新** — 既存テストの修正＋新スキルのテスト追加
6. **バージョン上げ** — pubspec.yaml version更新

- **後方互換性**: 法師で繰り返しクエストを作成済みの既存ユーザーのデータは、冒険者移管後も継続利用可能にする必要がある

- **Hiveマイグレーション**: Playerモデルの構造変更はないため、マイグレーション不要の見込み
- **テスト**: 215→? 既存の法師・侍関連テストは大幅修正が必要

## 六、付録：現行コードの関連箇所

### TaskCompletionService.complete() のスキル分岐
```dart
// 法師: 繰り返しタスク → 冒険者に移管予定
if (player.canUseSkill(Job.cleric) && task.repeatInterval != RepeatInterval.none) {
  task.lastCompletedAt = DateTime.now();
}

// 侍: コンボボーナス → 底力ボーナスに変更予定
if (player.canUseSkill(Job.warrior)) {
  player.comboCount++;
  final comboBonus = player.comboCount * 10;
  expGain += comboBonus;
}
```

### 疲労補正
```dart
final fatigueMultiplier = FatigueService.fatigueMultiplier(player);
expGain = (expGain * fatigueMultiplier).round();
coinsGained = (coinsGained * fatigueMultiplier).round();
```

### 冒険者スキルの現在の使われ方
```dart
// player.dart: questSlots - 冒険者マスター時は冒険者Lvの枠数を使用

// これらは「繰り返しクエスト登録」の権限制御に転用可能
```
